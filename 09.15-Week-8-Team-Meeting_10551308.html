Decision about code review:

Code review is a critical process in software development that helps improve code quality, find and fix issues, and ensure that the software meets its requirements.

We decided to do the peer code review, one person responsible for one epic.

As,

  • Use peer code review for larger, collaborative projects or critical code that will be used in production.

  • When you need diverse perspectives and want to catch as many issues as possible.

  • It's beneficial for maintaining coding standards and ensuring adherence to best practices.

  • For complex or high-stakes projects where the cost of errors is high, peer review is essential.
