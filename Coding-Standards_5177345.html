GitHub Branch Usage Standards

Whenever you start the development of a new feature, please follow the following steps:

Create a New Branch

Before beginning the development of a new feature, create a new branch from the latest main branch. The naming convention for the new branch is as follows:

<story>-<featureName>-<developerName>

For example:

Photos-ShowingPhotoOnMap-Bob

Write the Code

Write your code on your individual branch. Ensure that the code aligns with the projectâ€™s coding standards and style guidelines.

Commit the Changes

Regularly commit your changes as you write your code, with detailed messages describing the changes made.

Create a Pull Request (PR)

Upon the completion of feature development, create a pull request to propose your changes to the other team members for review.

When creating a pull request, you should provide a detailed description that illustrates the changes made and the new functionalities added. The PR description should include:

  • A brief description of the feature

  • How you have implemented it

  • Any other pertinent technical or background information

Code Review

Other team members will review your code. They may ask questions or suggest improvements.

Merge the Code

Once your code has been approved by the team, it will be merged into the main branch.

GitHub Commit Message Standards

Commit Message Format Standard

To maintain consistency and clarity in our codebase, please adhere to the following commit message format:

<featureName>: <shortDescription>

For example:

ShowingPhotoOnMap: Add highlight functionality on photo click

Detailed Commit Message

For more significant changes or when more context is necessary, use the following format:

<featureName>: <shortDescription>

- Detailed point 1
- Detailed point 2
- ...

Other Standards for Writing Commit Messages:

  1. Be Concise: The short description should be brief yet descriptive. Avoid vague messages like "fixed bug" or "updated code".

  2. Use Present Tense: Frame your commit message in the present tense, like "Add" instead of "Added", "Change" instead of "Changed".

  3. Avoid Punctuation in Short Description: Do not end the short description with a period.
