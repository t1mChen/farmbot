Overview

Over the third sprint, our team has successfully completed the User stories of all must have cases, should have cases and also one of the two could have case. Following the development of the cases, user acceptance tests and user experience tests are conducted. Due to the complexity of creating and conducting automated tests and restriction on work load of virtual machines, our team has also successfully conducted automated integration testing and several unit tests. We’ve also made several adjustments and refinement to previous artefacts .

Retrospective

Things we did well -

  • Good planning and task management: Group member intake different subjects in this semester, and they have no time to contribute while they have assignment of another subject due, so in the sprint planning session, we record the due date of other assignment, and allocate task base on group members' available time, to make sure everyone can complete their work in time.

  • Allocation of tasks: In this sprint, we set up table not only for the coding task but also allocate the code review task and test tasks to each group members, this make our team worked more efficiently and ensure the transparency.

  • Effective communication: In this sprint we set up detailed plan for team meeting, have more communication in slack and included a communication metric in confluence, all of our group member are able to cooperate with each other and ask other for help. Great!

  • We split our decision into subpages in this sprint, this provide a clear way for team members to understand our project and their task. Making the team more productive.

  • Jira management:

  1. we added a new column in Jira called review,

  2. In this sprint, we checked Jira and update it almost everyday, and now the burn down chart improved a lot.

  • Coding standard

  • Stable virtual machine

  • Efficient

Task estimation -

  • Finished 12/13 functional requirement.

  • All user acceptance test and code review for finished user story

  • Update Architecture design

  • User experience test and polish the system regarding to that result

  • Automated integration test half way done

  • Ethic & security report

  • Final presentation

Things that can be improved -

  • Better time management: Our progress is alright now, but in the planning, we are supposed to finish all functional requirement as now we only complete 12/13, this also block the progress of code review and tests. As well, we planto finished coding in the first week, but actully it is finished at very end of the sprint, we have no time to polish the system in sprint 3, and have to bring it in to next sprint.

  • Need to pay more attention to non functional requirement.

  • User oriented: In the user experience test, we found that we still have a lot of things to do. In the process of conducting user experience tests, we have noticed that we placed too much emphasis on technical aspects during web development and overlooked the user's experience while using the demo. For instance, the aesthetics of the webpage and providing users with a well-defined navigation path. This inspired us that our focus will primarily be on enhancing the user experience in the future.

  • Unexpected internet outage

Steps for improvement -

  • Non functional requirement:

Define test cases and testing strategies specifically for nonfunctional requirements. This may involve performance testing and usability testing, among others.

Implement monitoring and measurement tools to continuously track the system's performance and other nonfunctional aspects in real-world usage. This can help identify issues and areas for improvement.

  • User oriented: Add a table to record tasks from user experience test.
