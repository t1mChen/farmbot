9.08

Code Investigation:

Potential development method for simulating the actions by a “Farmbot device”. Under investigation of the current codes, when user is logged in as demo and attempting to access features that cannot be done under current development, the frontend side checks if it is a demo usage by involving with function maybeNoop(), where there is a function named ForceBotOnline().

Hence to complete the demo features of movement, we have decided to create a predefined user data with storage of information of positions. When the using it from demo, it records the data of change in movement and dispatch to places involving displaying the new location data, using customized functions.

For the photos tab, we plan to continue investigate the fake state contents and tests provided, to see how we can simulate and store all of the photos taken file, and activate the locked functions.

US 09

User story US09 is about making a tour guide to introduce the demo， because we already have the FarmBot demo, with its guide we've decided not to create a new tour guide. We realised that users had trouble noticing the guide when they entered the demo. To solve this issue our plan is to enhance the existing guide making it more user friendly and helping users gain an understanding of FarmBot in an intuitive way. We're also adding introductions to highlight features that demonstrate how FarmBot can greatly assist with planting. Our goal is to enhance the user experience by providing informative guidance within the existing demo. This way users can easily access information, about FarmBots capabilities. Understand how it benefits planting and gardening.
