Overview:

Within the second sprint, we’ve decided to begin our first phase of modification of the existing. This is the most essential sprint of the project since we need to decide how should we split the epics into subtasks, estimation of the duration of tasks, and assign to each individual group member. Throughout this sprint, we expect to begin to work on all must have, that is, the most important features that the Farmbot App must have to attract potential customers while they are playing around with the demo account.

The following tasks are proposed by our team members, which is expected to be carried out by the end of the sprint.

  • Modify user story base on client's feedback.

  • Break the user stories into multiple tasks and assign difficulties to each task. Allocate tasks to group member and build the system all together.

  • Start coding, plan to finish all “must have” tasks and attempt some of “should have” tasks.

  • Do the code review to ensure everyone follow the code standard

Task allocation and difficulty estimation for sprint 2:

Following the user stories to be developed, there are also subsequent steps that matches with each user story

  • developer self testing: functionality

  • code review check with standards: Code Review Checklist Template

  • unit testing with standards: Unit Testing Plan

Once the initial testing passes, there are also additional steps:

  • process pull request and merge,

And then followed by integration testing.

Here, we have discussed the potential role allocations for the major coding tasks, and the specific role allocation / progress are updated on JIRA: https://jiahaoshi.atlassian.net/jira/software/projects/HAR/boards/3/backlog
