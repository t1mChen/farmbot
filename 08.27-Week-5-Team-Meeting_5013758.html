Decisions

Add communication diagram:

Despite having various tools at our disposal to aid in teamwork and the agile process, we still experience some inefficiencies in collaboration. To resolve this issue, our team decides to use RACI matrix to document roles each person will play in different tasks. With this method, everyone involved can understand their responsibilities and the responsibilities of others and know the person to turn to when encountering problems, which prevent confusion and greatly promote collaboration.

Add new features:

  • Tour Guide feature: we find that the current tour guide lacks sufficient clarity, so we decide to modify it to make it more clear. This is a should have requirement since a good tour guide can improve users experience as well as maximise the value that users can derive from the app, increasing the likelihood of them to buy a product.

  • Support multiple models: we decide to make the demo support multiple models so that user can compare characteristics and limitations of them. This is a could have feature since it might be technically complex.

  • Advertisement: System can pop up farmbot ads to users. However, this feature might disrupt the user's browsing, making them unsatisfied. So we label this as could have.
