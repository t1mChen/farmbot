Check this page for more info: https://developer.farm.bot/v15/docs/farmbot-software-development and https://developer.farm.bot/v15/docs/web-app/frontend
4. If you build a Dashboard, I'll focus on an API that monitors changes on a target csv file for the e-nose, and for the thermal camera, assume that you monitor images in a folder, as different cameras have different standards. For the e-nose, Talking with the colleagues from Agriculture, they mentioned that it would be great to automatically recognise when the signal is in the "statble state". They mention that this happens after x seconds of holding the e-nose still over a plant, and then the sensors signal has a bell-curve shape. You could design a dashboard that loads the data and automatically recognizes the calibrated regions of the time series. They will send me the data soon, the team is currently in Europe, coming back on the 28th of Aug. If you are interested in building a ML pipeline, take a look if you can use datasets from Kaggle that contain ground truth data. You can certainly find datasets for FLIR cameras, but you may want to explore other datasets in the agricultural or plant space.
