Automated test bugs fixing

There are 6 bugs reported by the automated test from the .

Fail 1: This bug is caused because we replace entering message with our team name, which is inevitable and can be ignored.

Fail 2: Fixed by using the replace method instead of just concatenating the rotation message.

Fail 3, 4, 5 (photo related issues): These 3 bugs are caused due to the reason that we are not connected to a real camera. So it is hard to fix and will cause a lot of issues if we try to modify the camera-connection-related codes. So we put these bugs aside for now.

Fail 6: This bug is caused because there are two flippers (embedded and overlay) that if we update the state of one flipper, the other one does not update accordingly (two flippers should reflect each other' change)

Trial: lifting the states of flipper up to the simulated local store so that two flippers are connected to one state.

Outcome: fail 6 is no longer detected when manually testing this feature. However, automated test still reports this issue. We will try more possible solutions before the product is submitted.
