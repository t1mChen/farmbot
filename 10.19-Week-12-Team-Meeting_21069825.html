 Decisions

User experience test

Based on the results of the user experience test, we have summarized some area that need improvement. We have recorded more detailed information inUser experience testing

  1. Add a clear start point

User do not know how to manipulating the demo begin with, and they had to click around to find out what is available.

From all features in nav bar, we decided to highlight two main button, one is planting, lead the user try to plant(move) the plant icon themselves, and another one is photo button, engage the user to try out new add on features.

As well, we plan to all a page in the tour guide to lead the user begin with planting panel.

  1. Highlight the Robotic arm

One of the key feature of the farmbot is user can control their garden remotely with the robotic arm. However, the position of robotic arm is hard to seen, especially when the background image is displayed. n this case, we decided to make the position point more easier to be seen.

  1. Match the plant icon to the background image

We displayed the photo of plants in the demo interface but is doesn;t match the plant icon! So it is necessary to move the icon match the background image to make the demo feels more realistic.

  1. Search

We mainly focus on the photo panel in this project, but now is time to pay some attention to other aspect. When user try the planting panel, they hesitate about what should do, to give them better experience, we decides to add some details in the photo panel.

  1. Soil height detection

Should have clear response when the user measure soil height.

  1. Tour guide

Almost all user ignore the tour guide or accidentally close the tour guide when they enter the demo. We need rescale the tour guide window as well add more visual aids to attract user read through whole guide, this will help them have better experience with the farmbot demo.

  1. Feedback

Need to provide some previous comment to make the it more realistic.

  1. Timelaspe component

Users were unable to change photos using the timeline and spent a lot of time looking for ways to change photos, we need to make photo change when user sliding timeline/ give user a instruction about how to change photos.

Automatic Integration test

When writing test code, we need to recreate the environment that the program encounters during actual operation. This often involves handling many props and states, with the code frequently scattered across various files. Moreover, these files might contain additional related functions. Such clustering of files and code can lead to unpredictable side effects when employing mocking features of testing frameworks like Jest.

For example, a demo detection function named "forceOnline" Utilizing the mock feature on this functions can result in a lot of errors in other unrelated tests, making it exceptionally difficult to isolate the function being tested and thereby challenging to test a feature's completeness.

At 10.20, we only completed automated test for epic 2, and met obstracle when trying to test rest of cases.

As we have code review and manual unit test already finished, the current system is ok to run and further tests is not necessary. We decides to move our attention to polish the system rather than struggle with the integration test.
