HarvestX : Sprint 1 Planning

Created by Tianxi Chen, last modified by Xinxin Guo on Aug 26, 2023

This sprint is the setup of the entire project phase. It is significant to contact and discuss with the client to carefully confirm the details of project to be developed that is valuable and significant. Effectively, regarding to gathering the specific requirements and come up with an agreement. Since we are planning to do a contribution to the open source project. Later in this sprint, the UI and architectural design of the dashboard will be reviewed. The practicability of the proposed features / projects will be confirmed. And potentially, to start developing on the project later in the phase.

  • Set up roles for team members

  • Decide on tool stack

  • Design system using necessary diagrams

  • Set up IDE and VM

At end of the sprint, we are expecting to gather all of the agreed functional and non-functional requirements with the client. Understand the UI and architecture design, and progress on the coding and development part of the project.

We will accomplish this goal by organising frequent team discussions focusing on the agile development ceremony, assign relevant roles to team members that have expertise in each respective areas. Also, check with progress and maintain communication with client in a timely manner to ensure that the team is actively contributing to the task.
