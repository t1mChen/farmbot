Deployment & Handover:

Deployment:

unlike in a traditional project, we’re contributing and developing new features and functions towards an open source project. Hence we should follow the provided development standards by Farmbot official documents and github page.

Therefore, we have decided to deploy our service using strategy of IaaS using Goggle Compute Engine. This effectively starts a virtual machine instance running the ubuntu system, which is the correct method as acknowledged by the official Farmbot Web App development github.

With this deployment strategy, we can develop function that can be easily adapted towards the original development file.

Handover:

For our scope of project, our ultimate handover would just be a pull request from the staging branch towards the original github of Farmbot Web App.

To manage our handover, we will develop a table of contents, including all relevant files and artifects, a changelog to record the functions that has been added to the existing system. Also, a readme file would also be essential for the client’s benefit in understanding.

Details of handover will be confirmed soon as we approaching the handover stage.
