HarvestX : US-06 2.3 Resetting Code Review

Created by Jiahao Shi, last modified by Xinxin Guo on Oct 03, 2023

  1. Functionality

  • functionality is achieved
  • consistency with requirements and expectations
  • edge cases properly handled

  1. Readability

  • naming conventions
  • code format and organisation

  1. Structure

  • code follows existing architecture design
  • modularity of code is maintained: well organized function exporting and importing

  1. efficiency

  • memory well optimized: not adding much complexity
  • appropriate data structure: Used the original data structure to maintain consistency

  1. code reuse

  • correct management of dependencies: correct import
  • no duplicate codes: created a return home function for reuse

  1. Coding standards

  • coded appropriately: reasonable branch name

  1. Comments

  • clear comments to explain the codes: not enough comments
